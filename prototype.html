Responsive ads

About this project

This is intended to be a proof-of-concept for context-aware ad serving in a responsive web site. The 'context' we're concerned with is a set of arbitrarily defined pixel width ranges. By comparing an agent's current viewport width to a set of contexts in a configuration object we can select the most appropriate context, yielding a list of ad positions for to the agent's current state.

We are using Open AdStream (OAS) for testing, because it is the ad service used by The Boston Globe.

How it works

An agent's width is measured when the page loads using the clientWidth property, which should gather the agent's current viewport dimensions, as opposed to the devices' screen width. Next, a configuration object is looped through and compared to the clientWidth value. The configuration object defines pixel ranges in which a group of ad positions should be used. When a context is selected, it is feed to the OAS property listpos. OAS then renders those ad positions.
